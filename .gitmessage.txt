


# === 팀 커밋 메시지 규칙 ===
# Header ===
# <type>: <subject>
#   - 필수 작성
#   - type: 소문자만 허용 (!BREAKING CHANGE, !HOTFIX 제외)
#   - subject: 대소문자 시작 제한 없음 / 50자 이내 작성
#   - Header와 Body 사이 한줄 공백 필수
#
# Body ===
#  - 선택 작성
#  - 변경 이유, 해결 방법, 참고 사항 등
#
# Footer ===
# 유형: #이슈 번호
#  - 선택 작성
#  - 이슈 트래커 유형 : Fixes (미해결) / Resolves (해결) / Ref (참고) / Related to (연관)
# ============================
# === 주요 타입 ===
#  - `feat`: 새로운 기능
#  - `fix`: 버그 수정
#  - `design`: UI 스타일 변경 (마크업/CSS 수정 등)
#  - `!BREAKING CHANGE`: 호환성을 깨뜨리는 변경
#  - `!HOTFIX`: 긴급 패치
#  - `style`: 코드 포맷 변경 (동작 영향 없음)
#  - `refactor`: 리팩토링
#  - `docs`: 문서 변경
#  - `test`: 테스트 추가/수정
#  - `chore`: 기타 변경
#  - `rename`: 파일/폴더명 변경
#  - `remove`: 파일/코드 제거
